# 项目启动

当进入这个仓库的新会话，或用户要开始一个新任务、继续上次任务、接手已有工作、先理解当前项目状态时，使用这个 skill。

如果用户只是简单打招呼、测试助手是否在线、测试网络是否通畅，或发送任何信息量很低的探活消息，但当前上下文已经明确位于本仓库，也必须先完成最小启动流程：读取 `CLAUDE.md` 和 `progress.md`，再继续响应。

## 硬规则

- 这个 skill 是本仓库会话的默认前置门槛，不是可选参考
- 只要当前工作目录或上下文明确落在本仓库内，就先执行最小启动流程，再回复用户
- 不要因为用户消息很短、像寒暄、像探活、像网络测试，就跳过启动流程
- 只有在用户明确表示要纯聊天，且话题与仓库工作无关时，才允许跳过
- 如果已经读过 `CLAUDE.md` 和 `progress.md`，本会话后续无需重复读取，除非用户要求重新同步项目状态

## 启动流程

1. 先读 `CLAUDE.md`
2. 再读 `progress.md`
3. 只在需要时补充读取：
   - `PRD.md`：功能范围、版本规划、MVP 取舍
   - `design.md`：界面、布局、样式规范
   - `growth.md`：产品决策背景、复盘和方法论
4. 只检查和当前任务相关的代码
5. 在开始编辑前，先简短说明当前状态和下一步动作

## 触发示例

以下输入在本仓库中都应触发本 skill：

- `hi`
- `你好`
- `在吗`
- `继续`
- `看看现在做到哪了`
- `先帮我看看这个仓库`
- `我测一下网络`
- `先别写代码，读一下项目现状`

## 约束

- 把 `CLAUDE.md` 和 `progress.md` 视为主要交接文件
- 不要用 Codex 专用约定替换现有共享工作流
- 额外读取的上下文要克制，不要一次性扫所有文档
- 如果这次工作显著改变项目状态，应更新 `progress.md`
- 如果符合 `CLAUDE.md` 里的复盘规则，应提示更新 `growth.md`
