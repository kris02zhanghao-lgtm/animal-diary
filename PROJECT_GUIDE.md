# 项目协作说明

这个仓库同时兼容 `claudecode` 和 `Codex`。

## 共享上下文
- `CLAUDE.md`：长期稳定的项目说明、约束、环境变量名、协作规则
- `progress.md`：当前进度、下一步、已知问题、交接信息
- `PRD.md`：产品范围和版本规划
- `design.md`：界面和视觉规范
- `growth.md`：复盘、决策原因、可复用经验

## 建议启动顺序
每次接手这个项目，建议按下面顺序读取：

1. `CLAUDE.md`
2. `progress.md`
3. 如果任务涉及版本规划、功能范围，再读 `PRD.md`
4. 如果任务涉及界面或样式，再读 `design.md`
5. 最后再读相关代码

## 工具切换规则
- 只存在聊天里的内容，不会自动被另一个工具看到
- 写进仓库的文件改动，两个工具都能看到
- 如果希望跨工具无缝接手，优先把状态写进 `progress.md`

## 约定分工
- `CLAUDE.md` 继续作为项目总说明，不轻易改定位
- `progress.md` 继续作为当前执行记录
- `.codex/skills/` 只放 `Codex` 专用工作流，不替代上面两个共享文件
