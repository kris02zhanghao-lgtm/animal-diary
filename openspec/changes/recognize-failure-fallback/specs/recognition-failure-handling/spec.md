## ADDED Requirements

### Requirement: 识别失败时显示空卡片并允许保存
当 AI 识别失败或超时时，系统 SHALL 显示空白偶遇卡片模板（含照片、空的 title/species/journal 输入框），允许用户手动填充字段并保存记录。

#### Scenario: 识别失败后显示空卡片
- **WHEN** AI 识别失败（返回 error）或超时（无响应超过配置时限）
- **THEN** 系统显示空卡片（照片来自用户上传，其余字段为空），同时在卡片下方显示"❌ AI识别失败，请手动补充"提示

#### Scenario: 用户手动填充并保存
- **WHEN** 用户在失败卡片中填入 species（至少）且点击「保存到日志」
- **THEN** 系统调用 save-record API 写入记录（species、location、journal 等字段），保存完成后跳转回列表页

#### Scenario: 失败卡片中物种为空时保存禁用
- **WHEN** 用户未填写 species 字段且点击「保存到日志」
- **THEN** 保存按钮保持禁用状态（灰显），不提交 save 请求

#### Scenario: 用户放弃手动填充
- **WHEN** 用户未完成填充且点击返回箭头
- **THEN** 系统退出新建流程回到列表页，已上传的照片丢失（与识别成功后的行为一致，刷新重置）

### Requirement: 识别失败状态视觉区分
失败卡片与成功卡片在视觉上 SHALL 有明显区分，帮助用户快速识别失败状态。

#### Scenario: 失败卡片包含失败指示
- **WHEN** 显示失败卡片
- **THEN** 卡片或物种输入框附近显示 "⚠️ 手动填充" 或类似标签，使失败状态一眼可见

#### Scenario: 失败提示自动消失
- **WHEN** 用户开始编辑任何输入框（输入内容）
- **THEN** 识别失败提示可仍保留（不自动消失），或在卡片顶部收起，由设计阶段决定

### Requirement: 错误信息展示
识别失败的错误信息 SHALL 以非阻挡方式呈现，允许用户继续操作卡片。

#### Scenario: 错误信息位置
- **WHEN** 识别失败
- **THEN** 错误提示显示在「重新生成」按钮下方或卡片底部（不显示为全屏覆盖框）

#### Scenario: 多次识别失败不累积错误
- **WHEN** 用户点击「重新生成」多次，仍失败
- **THEN** 每次失败的错误提示覆盖前一次（不累积多条错误文本）
