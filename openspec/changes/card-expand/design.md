## Context

ListPage 当前用 Swiper coverflow 渲染折叠态卡片。需要在不引入新路由的情况下，实现点击菜单展开详情视图，并在详情和列表间切换。

## Goals / Non-Goals

**Goals:**
- 卡片右上角 ⋮ 菜单（展开、分享、删除）
- 展开后显示大图 + 完整日志，左上角 ← 返回按钮
- 详情视图右上角也有 ⋮ 菜单（分享、删除），为未来功能扩展预留接口
- 同一时间只有一张卡片展开

**Non-Goals:**
- 不做动画过渡
- 不支持在展开态编辑记录
- 不引入新的路由或弹窗库

## Decisions

**用 `expandedId` state 记录当前展开的卡片 id**
ListPage 添加 `expandedId` state（默认 null）。点击"展开"时设为该卡片 id，点击"返回"或"展开其他卡片"时更新 expandedId。天然保证同一时间只有一张展开。

**菜单用原生 dropdown（不引入新库）**
点击 ⋮ 后用 `position: absolute` 的菜单浮层显示，点击菜单项或点击外部时关闭。保持代码简洁。

**展开态在同一页面内条件渲染，不用路由**
展开内容直接追加在列表下方或浮层上，Swiper 轮播中的卡片和展开态共享 `expandedId` state，确保数据同步。

**删除只在菜单里，不单独放按钮**
无论折叠还是展开，删除都通过菜单访问，保持统一。删除后自动关闭菜单和展开态，刷新列表。

## Risks / Trade-offs

- [展开态在列表中占屏幕比例大] → 可以用固定高度或底部 sheet 样式，此处暂定条件渲染，后续可优化
- [菜单被下方内容遮挡] → 菜单用 z-index 确保浮在最上层
- [删除后 expandedId 残留] → 删除时主动 setExpandedId(null)
