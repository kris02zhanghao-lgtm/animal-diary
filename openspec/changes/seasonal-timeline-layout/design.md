## Context

当前 ListPage 渲染逻辑：直接 `records.map(...)` 输出单列卡片，按 `createdAt` 倒序（由 storageService 保证）。需要引入分组计算层，将 records 聚合为 `{ seasonKey, label, records[] }[]` 结构后再渲染。

## Goals / Non-Goals

**Goals:**
- 列表记录按季度分组，每组显示标题（如"2026年春"）+ 横向滑动卡片轨道
- 顶部 tab 切换 UI 占位（"时间线 / 图鉴"）,样式到位但点击不响应
- PRD.md 版本规划对调

**Non-Goals:**
- 不实现 tab 切换功能（v0.4 再做）
- 不新增图鉴页（v0.4 再做）
- 不改变卡片内容和删除逻辑

## Decisions

**分组计算放在组件内，不放 storageService**

分组是纯展示逻辑，与存储无关。在 ListPage 内用 `useMemo` 对 records 做分组计算，避免污染 storageService。

季度判断函数：
```
3-5月 → 春，6-8月 → 夏，9-11月 → 秋，12/1/2月 → 冬
seasonKey = `${year}-${season}`（如 "2026-春"）
```
分组后按 seasonKey 倒序排列（最新季度在最前），组内记录保持原有倒序。

**使用 Swiper.js + coverflow 效果**

引入 Swiper.js，使用 coverflow 模式实现中心焦点轮播：当前卡片在屏幕中心放大，两侧卡片缩小并部分隐藏。滑动时下一张卡片滑入中心并放大，带有缩放 + 旋转的 3D 视觉效果。

**卡片宽度调整为 260px 左右**

卡片保持竖版格式但宽度增加，上方图片（正方形裁剪）、下方信息行包含：🐾 物种名、📍 地点、日期。保留必要信息，点击卡片后可展开查看完整日志（由 card-expand change 实现）。

**tab 占位样式**

顶部两个 tab 按钮（时间线 / 图鉴），默认"时间线"为选中态（暖色下划线），"图鉴"为未选中态。`onClick` 为空，不绑定任何事件。视觉上像真实 tab，但功能未开放。

## Risks / Trade-offs

- [记录全在同一季度] → 只显示一组，横向卡片依然正常工作，无问题
- [季度分组标题占据垂直空间] → 页面结构从"纯列表"变为"分组列表"，整体更高，可接受
- [tab 占位让用户以为可点击] → 样式上"图鉴" tab 颜色略淡，视觉上暗示未激活；tooltip 或 badge 可后续加
