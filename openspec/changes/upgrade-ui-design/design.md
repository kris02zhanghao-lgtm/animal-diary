## Context

当前项目使用 React + Tailwind CSS，已有基础的温暖配色。UI 设计规范已在 `design.md` 中完整定义（色彩体系、圆角、阴影、动画等）。参赛需要将现有 UI 升级为高品质的统一风格。

## Goals / Non-Goals

**Goals：**
- 将整个应用升级为统一的动物森友会风格
- 确保所有页面和组件外观一致（颜色、圆角、间距、阴影）
- 保证交互体验流畅（悬停浮起、动画等）
- 本地测试验证无样式冲突或布局问题
- 确保移动端和桌面端都正常显示

**Non-Goals：**
- 不改造业务逻辑或功能流程
- 不修改后端 API 或数据模型
- 不增加新的 npm 依赖
- 暂不处理分割线、页脚等需要 SVG 的装饰元素

## Decisions

**决策 1：Tailwind + CSS 变量混合方案**
- 保留 Tailwind 的基础样式和布局工具
- 用 CSS 变量定义设计规范中的色值、圆角、阴影等
- 理由：Tailwind 内置的 shadow 不支持 3D 按压效果，需要用 `box-shadow` 手写；CSS 变量便于统一管理和快速调整

**决策 2：优先级：组件 → 页面布局**
- 先改造 Button、Card、Input、Modal 等核心组件
- 再改造 ListPage、NewEncounterPage 的整体布局
- 理由：组件是构建块，先统一好再组装页面，降低返工

**决策 3：兼容现有 Tailwind 类**
- 不删除或修改现有的 Tailwind 类，只在 `<style>` 中添加全局样式和 CSS 变量
- 理由：降低改造风险，避免破坏已有的功能

**决策 4：本地完整测试后再部署**
- 用 `npm run dev` 本地验证所有页面
- 检查移动端、平板、桌面的响应式效果
- 确认无样式冲突、布局错位、动画卡顿
- 通过后才推送到 Vercel
- 理由：避免线上出现样式问题

## Risks / Trade-offs

| 风险 | 缓解方案 |
|-----|--------|
| 改造涉及多个文件，可能遗漏某些组件 | 按清单逐个检查，ListPage 和 NewEncounterPage 全页扫描 |
| 新的 CSS 变量与现有 Tailwind 样式冲突 | 先在开发环境测试，用浏览器 DevTools 检查最终渲染的样式 |
| 3D 按压效果在某些浏览器或设备表现不同 | 测试覆盖 Chrome、Safari、Firefox；使用 box-shadow 是标准 CSS，兼容性强 |
| 圆角或间距改动可能影响页面整体布局 | 用 grid/flex 布局抗性强，小幅改动不会破坏；优先测试关键路径 |
| 本地测试不完整导致上线后才发现问题 | 制定详细的测试清单，覆盖每个页面、每个交互状态 |

## Migration Plan

1. **第 1 阶段 - 定义全局样式**
   - 在 `src/index.css` 中添加 CSS 变量（颜色、圆角、间距、阴影、动画）
   - 设置全局字体、行高、背景色

2. **第 2 阶段 - 改造核心组件**（按优先级）
   - Button（所有类型和尺寸）
   - Card（卡片）
   - Input（输入框）
   - Modal / ConfirmDialog（弹窗）

3. **第 3 阶段 - 改造页面**
   - ListPage - 列表布局、卡片样式、tab 样式
   - NewEncounterPage - 表单布局、上传区域、结果卡片

4. **第 4 阶段 - 本地测试**
   - 功能测试：上传照片、AI 生成、保存、删除
   - 样式测试：全屏、响应式、深色模式（如果有）
   - 浏览器测试：Chrome、Firefox、Safari
   - 设备测试：手机、平板、桌面

5. **第 5 阶段 - 部署**
   - 提交 git commit
   - 推送到 main 分支
   - Vercel 自动部署
   - 验证线上效果
