## 关键决策

### 决策1：在 prompt 中约束物种名，不做 AI 后置标准化

直接在 `api/recognize.js` 的 prompt 里补充说明：species 字段使用行业通用中文俗名，不加颜色/花纹修饰词。

**为什么**：不额外增加 API 调用，零成本，一次搞定。准确率上，因为约束的是 AI 自己输出，而不是后置纠错，效果更稳定。

**放弃方案**：后置再调用一次 AI 标准化——多一次网络请求和费用，且纠错准确率未必比约束输出更高。

### 决策2：修改范围仅限 prompt 文字

只修改 `api/recognize.js` 中 species 字段的描述说明，不改动 JSON 返回结构、不改前端代码。

## 实现细节

修改位置：`api/recognize.js`，prompt 中识别任务第1条：

**现在**：
> 识别图片中的动物物种（中文名，生活化称呼，如"奶牛猫"、"橘猫"、"三花猫"、"柯基"、"喜鹊"，黑白花纹的猫称为"奶牛猫"）

**改为**：
> 识别图片中的动物物种，使用标准中文俗名（如"橘猫"、"三花猫"、"奶牛猫"、"柯基"、"喜鹊"）。规则：只写物种名，不加颜色或花纹修饰词（统一用"橘猫"，不用"橘白猫""橘黄猫"；统一用"奶牛猫"，不用"黑白猫"）；同一物种始终用同一个名称。

## 注意点

- 历史记录中已有的变体名（如"橘白猫"）不会自动修正
- 用户可通过详情页编辑功能手动统一历史记录的物种名
