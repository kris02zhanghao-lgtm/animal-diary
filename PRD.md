# 动物偶遇图鉴 · 产品需求文档
> 版本：v0.3 | 更新日期：2026.04.09

---

## 一句话描述

一款用AI记录你在城市中偶遇小动物的个人图鉴app，拍照即可自动识别物种、生成趣味日志，越用越丰富。

---

## 要解决的问题

路上碰到可爱的小动物会随手拍，但这些照片散落在相册里，没有专门的地方记录、回顾和赋予意义。现有工具要么太学术（iNaturalist），要么太通用（相册/备忘录），缺少一个轻量、有趣、有情感温度的个人动物偶遇记录产品。

---

## 目标用户

喜欢随手拍小动物的城市年轻人（不是专业观鸟/自然爱好者，就是普通的"看到猫猫狗狗会停下来拍照"的人）。

---

### 🟢 F1：照片上传 + AI识别 + 日志生成（MVP）

**用户操作：**
1. 点击「记录偶遇」按钮
2. 从本地选择一张照片（不做拍照，避免移动端适配麻烦）
3. 点击「生成日志」
4. 等待AI返回结果，展示：
   - 识别到的动物种类（如"橘猫"）
   - 一段趣味偶遇日志
5. 用户可以在文本框里手动编辑日志 ← **待开发**
6. 点击「保存这次偶遇」→ 存入本地

**AI调用方式：**
- 调用OpenRouter API，默认模型：google/gemini-2.5-flash-lite
- 一次调用同时完成：物种识别 + 趣味日志生成
- 日志风格：城市观察站视角，"你"为主语，动物是主导者

**暂时砍掉的：**
- ~~拍照功能（用选图代替）~~
- ~~自动获取GPS定位（用手动输入地点代替）~~

---

### 🟢 F2：偶遇列表（MVP）

**展示内容（每条记录）：**
- 照片缩略图
- AI生成的日志文字
- 动物种类标签
- 时间（自动记录保存时间）
- 地点（用户手动输入的文字）

**交互：**
- 按时间倒序排列
- 点击一条记录 → 展开查看大图 + 完整日志 ← **待开发**
- 点击删除按钮 → 弹出二次确认弹窗 → 确认后删除该条记录（⚠️ 当前直接删除，待加确认弹窗）

**数据存储：**
- 全部存localStorage，无需后端 ✅ 已完成
- 图片转base64压缩后存储 ✅ 已完成

---

### 🟢 F3：基础页面框架（MVP）✅ 已完成

- 首页（列表页）：展示所有偶遇记录 + 右下角「+」按钮
- 新建偶遇页：上传图片 → 填写地点 → 生成 → 编辑 → 保存
- 星露谷像素风格UI

---

### 🟡 F4：偶遇地图（P1）

- 地图视图，每次偶遇是一个标记点
- 点击标记 → 展开该次偶遇的照片和日志
- 用的越久，地图越丰富，形成个人专属的"城市动物地图"
- 技术方案：高德地图JS API / Mapbox + 浏览器Geolocation API

---

### 🟡 F5：图鉴收集（P1）

- 按物种分类，展示"你遇到过的物种"
- 显示每个物种遇到过几次、在哪些地点遇到
- 图鉴完成度百分比（制造收集感）

---

### 🔵 F6：成就系统（P2）

成就示例：
- 猫色大师：集齐5种不同毛色的猫
- 夜行者：晚上10点后拍到动物
- 老朋友：在同一地点遇到3次以上
- 跨城旅行家：在2个以上城市记录过偶遇

---

### 🔵 F7：回头客识别（P2）

- 在相近地点上传外观相似的同类动物，系统提示"这可能是你之前遇到过的那只"
- 用户确认后自动关联，建立"回头客档案"
- 技术方案：同一地点（500m内）+ 同物种 + 图像相似度粗匹配

---

### ⚪ F8：分享卡片（P3）

- 一键生成偶遇分享卡片（照片+地点+日志+日期）
- 适合发朋友圈/小红书

---

## 技术方案

| 项目 | 方案 | 备注 |
|------|------|------|
| 框架 | React（Vite） | |
| 样式 | Tailwind CSS | |

## 风险与应对

| 风险 | 应对 |
|------|------|
| AI物种识别不够准 | 允许用户手动修正；识别粒度放粗（"橘猫"而非精确亚种） |
| 日志生成质量不稳定 | 精调prompt；允许用户编辑 |
| 使用频次低 | 成就系统制造主动寻找动机；图鉴完成度激发收集欲 |
| 回头客识别不准 | 定位为"可能是"而非"确定是"，用户手动确认 |
